## 1. The Prime Directive — Anti-Slop

You are building UI that looks like it was made by a **human designer at a top studio**, not generated by an AI.

Before writing a single line of CSS or JSX, ask yourself:
> *"Does this look like something an AI defaulted to, or something a designer chose deliberately?"*

If it feels like a default — change it.

## 2. Hard-Banned Patterns (Automatic Fails)

These patterns get auto-rejected. No exceptions unless the brief explicitly requests one.

### Visual
- No soft gradients as decorative backgrounds
- No glassmorphism / frosted panels as the default UI language
- No oversized rounded corners (max `10px` on cards, `8px` on buttons, `6px` on inputs)
- No pill-shaped buttons everywhere
- No colored glows or outer shadows for decoration
- No neon accents unless explicitly requested
- No `#000000` pure black — use off-black (e.g. `#0f0f0f`, `#111`)
- No `#ffffff` pure white — use off-white
- No dramatic drop shadows above `0 2px 8px rgba(0,0,0,0.10)`
- No gradient borders or gradient text on headings by default
- No fake product UI built from `<div>` rectangles in hero sections

### Layout
- No hero sections inside internal dashboards
- No metric-card grid as the first instinct for a dashboard
- No 3-column equal feature cards (the classic AI layout)
- No floating detached sidebars with rounded shells
- No overpadded layouts with dead space
- No sticky left rail unless the IA genuinely needs it
- No "control room" aesthetic unless explicitly asked

### Typography
- No `Inter` as default — use `Geist`, `Outfit`, `Cabinet Grotesk`, or `Satoshi`
- No serif as a default "premium" move — serif only if the brief names it
- No uppercase + letter-spacing eyebrow labels everywhere
- No oversized H1s that just scream loudly
- No mixed serif/sans combos as a shortcut to "editorial"
- No em-dashes (`—`) anywhere

### Content / Copy
- No "John Doe", "Sarah Chan" generic names — use real-feeling names
- No fake-perfect numbers (`99.99%`, `50%`, `1,234,567`) — use organic ones
- No filler verbs: "Elevate", "Seamless", "Unleash", "Revolutionize", "Next-Gen"
- No generic startup-slop brand names: "Acme", "Nexus", "SmartFlow"
- No section-number eyebrows: `00 / INDEX`, `001 · Capabilities`
- No decorative middle-dot (`·`) overuse — max 1 per line

### Animations
- No `window.addEventListener('scroll', ...)` — use Motion's `useScroll()` or GSAP ScrollTrigger
- No `useState` for continuous mouse/scroll values — use `useMotionValue`
- No bouncy spring animations on standard UI
- Only animate `transform` and `opacity` — never `top`, `left`, `width`, `height`
- All animations MUST respect `prefers-reduced-motion`

## 3. What Good Looks Like

Think: **Linear. Raycast. Stripe. GitHub. Notion.**

They don't try to grab attention. They just work.

### Sidebars
`240-260px` fixed width, solid background, `1px` border-right, no floating shells, no rounded outer corners.

### Buttons
Solid fills or simple borders, `8-10px` radius max, no pill shapes, no gradient backgrounds.

### Cards
`8-12px` radius max, subtle `1px` border, shadow max `0 2px 8px rgba(0,0,0,0.08)`, no floating effect.

### Typography scale
- Display: `clamp(2.5rem, 5vw, 4rem)`, `tracking-tighter`, `leading-none`
- Body: `14-16px`, `line-height: 1.6`, `max-width: 65ch`

### Spacing
Consistent scale: `4 / 8 / 12 / 16 / 24 / 32 / 48 / 64px`. No random gaps.

### Transitions
`100-200ms ease`. No bouncy animations. Simple `opacity` / `color` changes.

## 4. Three Design Dials

Set these before building any page. Override conversationally.

| Dial | Default | Range |
|------|---------|-------|
| `DESIGN_VARIANCE` | 7 | 1=symmetrical, 10=asymmetric/artsy |
| `MOTION_INTENSITY` | 5 | 1=static, 10=cinematic/physics |
| `VISUAL_DENSITY` | 4 | 1=airy/gallery, 10=cockpit/packed |

**Before writing code, state:** *"Reading this as: [page kind] for [audience], [vibe], dials: V=X M=Y D=Z"*

## 5. Color Rules

1. **First:** Use existing project colors (check `main/src/styles/` or `globals.css`)
2. **Second:** Pick from the palettes in `ai-rules/uncodixfy/SKILL.md`
3. **Never** invent random color combinations

Project accent: `#6C47FF` (violet-indigo) — use it, don't fight it.
Neutrals: Dark surface `#0f0f0f` / `#111827`, Light surface `#fafafa` / `#f5f5f4`.

## 7. Component Sources (use these, in order)

When building a new section or component:

1. **shadcn/ui via MCP** — live accurate components, no hallucinated props
   ```
   npx shadcn@latest add [component-name]
   ```

2. **21st.dev** — paste the component URL and run:
   ```
   npx shadcn@latest add [21st.dev URL]
   ```

3. **Efferd blocks** — heroes, pricing, auth, FAQ, etc.:
   ```
   npx shadcn@latest add https://efferd.com/r/[block-name].json
   ```

4. **Uiverse.io** — paste the link; the AI fetches and adapts the code

5. **awesome-design-md** — when the user names a brand style (e.g. "Linear", "Stripe", "Notion"), fetch from:
   ```
   https://raw.githubusercontent.com/VoltAgent/awesome-design-md/main/brands/[brand-name]/DESIGN.md
   ```

## 9. Spline Handoff Protocol

**STOP before attempting any 3D scene or complex interactive animation in code.**

When Spline is the right tool:
1. Stop. Do not write Three.js or canvas-based 3D from scratch.
2. Write a detailed Spline AI prompt for the user.
3. Specify export format: `@splinetool/react-spline` React component OR `<spline-viewer>` embed tag.
4. Wait for the user to paste back the exported code before continuing.

## 11. Project-Specific Rules (from memory)

- **Mobile-first CSS:** All base styles target mobile. Scale UP with `min-width` queries. Never `max-width` overrides.
- **Currency:** ₦ Naira only. No `$` anywhere.
- **No iOS input zoom:** All inputs globally have `font-size: 16px`. Never override below 16px.
- **No width shaking:** `overflow-x: hidden` + `scrollbar-gutter: stable` on `html` in globals.css.
- **Auth layout:** Split panel — dark brand left, light form right. Min-width 900px shows brand panel.
- **Shell CSS:** Has two `.sidebar` definitions — do NOT merge them, risk of breakage.
