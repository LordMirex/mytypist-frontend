# Landing Page Comparison: The Editorial Gateway

## High-Level Summary
- **Vision**: The landing page must be an **Editorial Gateway** to the Document Operating System. It should feel like professional publishing software (Adobe, Notion) rather than a generic SaaS mockup.
- **Goal**: High-fidelity conversion via **Live Generation Flow**.

## Extraction Recommendations for 'main'
1. **Layout**: Use **Document-Centric Split Layouts**. Left for precision copy; right for a live-generation document preview.
2. **Interactive**: Port v2's search and previewers but redesign them to be **Operational Tools** rather than "demo" components.
3. **Workflow Timelines**: Replace "Features" cards with a horizontal timeline of the document pipeline (Template -> Fidelity Check -> Operational Approval).
4. **Copy**: Eliminate generic startup buzzwords. Use authoritative, technical copy focused on **Formatting Fidelity and Structure Preservation**.

## Visual Standards
- **Typography-First**: The document itself is the visual center. No glowing buttons or stock photos.
- **Whitespace Rhythm**: Professional spacing that communicates layout confidence.
- **Conversion Flow**: Select -> Fill -> Preview -> Account required for Final Fidelity Download.
