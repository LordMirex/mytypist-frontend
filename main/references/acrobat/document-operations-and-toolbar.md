# Acrobat — Document Operations & Toolbar

Source: adobe.design blog + developer.adobe.com + clemson.edu accessibility guide

## What
"Megaverbs" concept: consolidate multiple workflows under single action verbs (Edit, Sign, Convert). Three interface views: Home, Tools, Document. Document view: tabbed interface for multiple PDFs with customizable toolbar.

## Why It Works
- "Megaverbs" changed design from individual tasks (organize pages, edit text, add images) to entire workflows (Edit, Sign, Convert). Users think in outcomes, not actions.
- Three views reduce complexity: Home (recent files + discovery), Tools (all capabilities catalogued), Document (focused work)
- Customizable toolbar: right-click to add/remove tools per user preference. Power users build their own layout.
- Quick Tools moved to left side (new Modern Viewer) — closer to document content reduces mouse travel
- Page thumbnails moved to right panel — consistent with natural reading flow (content left, navigation right)

## Key Details
- Tools categorized by function: Edit, Sign, View, Convert, Protect, Organize
- Document tab bar at top for multi-file workflow
- Context menus: right-click shows commands related to active tool/selection
- Right-hand panel: page thumbnails, bookmarks, navigation, zoom
- Quick action toolbar floats within document view (can be repositioned)
- "Save" button hidden until document is modified
- Dark UI: 9 element categories (gray text, white text, icon color, toolbar backgrounds, divider lines, hyperlinks, blue accent)

## Apply To MyTypist
- Document toolbar as megaverbs: Edit / Preview / Sign / Share / Export
- Tab bar for multiple open documents
- Contextual right-click menus for document actions
- Right panel for page thumbnails and document navigation
- Toolbar customization: let power users pin favorite tools
- Hide "Save" until document has unsaved changes
