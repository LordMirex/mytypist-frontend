# PandaDoc — Competitor Analysis

Source: pandadoc.com, support.pandadoc.com, Zendikt, G2 reviews

## Positioning
Proposal + eSign in one workflow. $19-65/user/mo. Best for sales-anchored organizations (10-2,000 employees). Strongest proposal + eSign combo in market.

## Design & UX Pattern

### New guided flow (2024 redesign)
- Broke the old single-screen editor into step-by-step guided flow: CRM → Recipients → Content → Payment → Approval → Send
- Document preview stays visible on the left throughout
- User can "Skip" any optional step or "Open document" to jump to the editor directly
- Step indicator at top shows current position and total steps

### Editor layout
- Left: document preview (always visible)
- Right: contextual sidebar (fillable fields, recipients, etc.)
- New top menu bar: File / Edit / Insert / View / Format (traditional desktop app pattern)
- Floating toolbar with AI edit button (star icon) on text selection
- "+" button on page to insert content blocks before/after

### Dashboard
- Simple list view showing documents by stage: Drafts, Action Required, Waiting for Others
- Customizable tabs (viewed, expired, waiting for payment, etc.)

## Known Failure Points (What NOT To Copy)
1. **Step-by-step flow can feel rigid** — users complained it added clicks to what was previously a one-screen operation
2. **Pricing tier erosion** — removed features from lower tiers, alienated small business users. User reviews: "less value for same price"
3. **"The tool fights you with images and placement"** — document editor struggles with complex image positioning
4. **Expensive for small business** — despite aggressive SMB positioning, users find pricing prohibitive
5. **Complex pricing tables** required extra configuration even with templates

## What MyTypist Can Learn
- **The guided flow concept is good for complex workflows** — but must allow power users to bypass it (Linear's Cmd+K principle: fast path for experts)
- **Always-show-preview pattern** — left preview + right config is proven. Don't hide the document.
- **Step indicator** — clear progress visibility reduces anxiety in multi-step flows
- **"Skip" as a first-class action** — non-blocking design respects user autonomy
- **Top menu bar** — traditional but functional. Linear uses minimal top chrome. MyTypist should use Acrobat's megaverb toolbar instead.
